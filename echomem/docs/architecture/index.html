Architecture

三家本地 Agent，
共享一颗会反思的本地大脑。

echomem 是 Claude Code、openclaw、hermes 的本地记忆与衍生物中枢。SQLite + 本地小模型，离线可跑，schema 与云端 dbay 同源——demo 不靠云、跨 Agent 不丢上下文、记忆衍生物在后台默默生成。

问题域

为什么要有一个本地中枢

日常在 Claude Code、openclaw、hermes 之间切换。同一个事实、同一份 skill、同一段会话上下文，每次切 Agent 就要重讲一遍。云端 dbay 解决了"持久化"，但解决不了"本地 demo 离线、跨 Agent 共享、衍生物在本地小模型上跑"这三件事。

01

跨 Agent 不互通

三家本地 Agent 各自一套记忆，切换工具就丢上下文。同事问的同一个问题，要在三处分别回答。

02

云依赖太重

客户/同事 demo 必须有外网，离线场景受阻。云端 dbay 适合长期持久化，但不该是 demo 前提。

03

衍生物没本地化

云上 pipeline 与 PG schema 紧耦合，本地小模型（gemma 4B、qwen3-embedding）跑不起同样的衍生物提取。

04

研究通道未铺

"用记忆预测 LLM 输出长度"对 MaaS 调度的价值还没验证。需要一条侵入性低的本地通道做实验。

分层架构

单进程 daemon · 多 transport 适配

FastAPI daemon 跑在 127.0.0.1:8473。MCP / HTTP / CLI 三种入口共享同一份 StorageDriver 抽象。Memory 是基底，衍生物（时间流 / 树 / 图 / 程序性）是 store 之上的视图，由后台 worker 异步填充。

层职责关键模块技术选型
Clients 三家 Agent + Dashboard，按各自原生协议接入 Claude Code (MCP) · openclaw / hermes (HTTP) · Dashboard SPA stdio · HTTPS · WebSocket
Transport 把客户端协议转成 daemon 内部调用 MCP shim · OpenAPI router · CLI adapter FastMCP · FastAPI
Daemon API + 后台 worker pool + 存储驱动抽象 Memory · Derivatives · Context · Sessions API · 5 workers Python 3.11 · FastAPI · uvicorn · asyncio
Storage 本地优先；schema 与云端 dbay 同源 SQLite (vec + FTS5) · Blob FS · CloudDriver (可选) sqlite-vec · FTS5
AI 本地小模型，所有衍生物都在本机生成 embedding · summarize · entity 抽取 Ollama · gemma4:e4b · qwen3-embedding
衍生物

四种组织方式 · 一份 store 之上的多副视图

每条 ingest 进来的记忆，会触发后台 worker 生成最多四种衍生物。它们共享同一个 store，按不同维度组织——Agent 检索时根据上下文需要选择性查询。

流

时间流（Episodic）

同 Agent + 30 分钟内 + 主题 cosine ≥ 0.7 → 同一个事件。无需 LLM，纯向量聚合。

树

分层摘要（L0/L1/L2）

L0 ≤ 100 tok 标题级、L1 ≤ 500 tok 段落级、L2 原文。短文本自动跳过 L1。

图

实体关系

(主语, 谓语, 宾语) 三元组。置信度 ≥ 0.7 入图，其余进 pending 表等审核。

序

程序性（Skill）

从 ~/.claude/skills 等目录 import frontmatter。description 做 embedding，按上下文 surface。

设计原则

六条不动摇的取舍

Local-first

本地是主战场。备份就是 cp ~/.echomem。云端 dbay 是可选挂载，不是默认依赖。

同源不重写

schema 与 API 概念尽量与云端 dbay 兼容。StorageDriver 抽象同时支持 SQLite / PostgreSQL / Cloud。

存储抽象层

所有上层只见 StorageDriver 接口。切换底层不动业务代码——这是同源策略的兜底。

小模型友好

衍生物 pipeline 对 gemma 4B 量级模型留降级路径。置信度低不污染正式数据，进 pending 表。

零云依赖默认

demo 必须在没外网时跑通。云挂载是用户主动行为，需要显式开启。

Hook 不入侵

Insight Track 等可选能力通过 Agent 框架 hook 接入，不修改 Agent 业务代码。

"echomem 不是一个新东西。它是把 dbay 已经验证过的那些概念——记忆、衍生物、AgentFS——挪回本地，让一台 MacBook 也能体面地服务三家 Agent。"
