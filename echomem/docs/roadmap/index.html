Roadmap

分阶段交付，不做抛弃型方案。

瞄准"三家 Agent 共享本地大脑"这一目标，分五个阶段抵达。每阶段独立可用、不依赖未来阶段；上一阶段的产出是下一阶段的前提。

当前状态（2026-05-06）： Phase 1（存储 backbone）与 Phase 2（衍生物 pipeline）已交付并 e2e 验证通过。Phase 3（Context API + Blob）的实施计划刚写完，进入实现期。Phase 4（Vue Dashboard）与 Phase 5（Onboarding 一键安装）待启动。
阶段一

Backbone — 一颗能用的本地记忆 store

把云端 dbay 的 Memory API（ingest / recall / list / get / delete）剥到本地。SQLite + sqlite-vec + FTS5 三件套，配 Ollama 嵌入。必须能跑、能查、能在没外网时给三家 Agent 提供基础记忆服务。

阶段二

Derivatives — 衍生物在后台默默生长

每条 ingest 触发后台 worker pool 做 4 类衍生物。价值不在某一类衍生物，而在它们共享同一份 store——Agent 可以按上下文切换查询视角，省 token、跨 Agent 不丢线索。

PHASE 2 · 2026-05-01
Derivatives
~6 个工作日
已交付

核心交付

  • m002 迁移：9 张衍生物表（event / summary / entity / triple / skill / task / dead_letter / ...）
  • SummarizerWorker：gemma 生成 L0 / L1 / L2 三级摘要
  • EntityExtractorWorker：三元组抽取，置信度 ≥ 0.7 入图，否则进 pending
  • TimelineWorker：30 min 时窗 + cosine 主题聚合（无 LLM）
  • WorkerPool：asyncio queue + 重试 + dead_letter 兜底
  • 4 个 derivatives 端点：timeline / tree / graph / skills
  • Skill importer：扫 ~/.claude/skills 的 frontmatter，按上下文 surface
  • ReflectorWorker 占位（P3 之后再 wire LLM）

验收

e2e：ingest 后 summary / event / triple 三类衍生物都正确落地；query_subgraph 边去重已修复；外部 Ollama 超时不再阻塞 worker pool。

阶段三

Context API — 不止文本，还有 URL/PDF/repo

把云端 AgentFS 的关键能力剥本地：add_url / ls / read / write / mv。原始文件按 sha256 存 FS、解析后内容进 memory，给 Agent 一个能"指向外部世界"的引用层。

Insight Track — 输出长度预测

独立于主路线的研究子项目。用 echomem 已有的记忆做特征，预测下一次 LLM 输出长度，给 MaaS 调度提供决策依据。研究通道靠 Agent hook 接入，不侵入主代码。先验证假设，再决定是否进 P1 接 MaaS。
