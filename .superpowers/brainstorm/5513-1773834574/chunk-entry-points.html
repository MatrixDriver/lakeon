## RESTful API 设计原则

API 的设计应遵循资源导向的架构风格，每个端点代表一个资源或资源集合。

核心原则：

1. 使用名词而非动词命名端点

2. 使用 HTTP 方法表达操作语义

3. 版本化 API（如 /v1/resources）

4. 使用合适的 HTTP 状态码

5. 支持分页、过滤和排序

每个资源应有唯一的 URI 标识，客户端通过标准 HTTP 方法（GET, POST, PUT, DELETE）操作资源...
