代码来源：选哪种方式？

用户的 Python 代码如何进入系统——这决定了整个创建页的交互核心

✏️

A. 内联代码编辑器

在浏览器里直接写脚本，类似 Glue Script Editor

Script Editor
# main.py — 在浏览器直接写 import pandas as pd import os path = os.environ["DATASET_PATH"] df = pd.read_parquet(path) df = df[df["score"] > 0.8] df.to_parquet( os.environ["OUTPUT_PATH"] )
  • 立即上手，无需预先上传文件
  • 适合短脚本（< 200 行）
  • 未来可接 AI 直接生成代码
  • 大项目（多文件）不适用
  • 无本地 IDE 的自动补全
适合技术用户 AI辅助友好
📦

B. OBS 代码包路径

用户把代码上传到 OBS，填写路径。容器启动时自动下载执行

代码来源
OBS 路径 obs://my-bucket/jobs/etl-v2/ 浏览
入口命令 python main.py --mode prod
容器启动时：obs cp obs://... /app/ && python main.py
  • 支持大型多文件项目
  • 代码版本管理在用户手中
  • 与本地开发工具链结合
  • 需要先手动上传代码到 OBS
  • 首次使用摩擦较大
适合大型项目 CI/CD 友好
🔀

C. 两种方式都支持

用 Tab 或 Radio 切换：内联脚本（快速）或 OBS 路径（项目）

代码来源
✏️ 内联脚本
📦 OBS 路径
import os, pandas as pd
df = pd.read_parquet(os.environ["DATASET_PATH"])
# 继续编写...
✨ 未来：AI 生成代码
  • 快速脚本和大项目都覆盖
  • 内联模式可对接 AI 生成
  • 类 Glue Script Editor 体验
  • 开发量稍多（两套路径）
  • MVP 可先只做内联，后加 OBS
灵活 AI辅助友好 推荐

💡 建议：选 C，但分两阶段

MVP：先做内联代码编辑器（Monaco Editor，类 Glue）。用户直接在浏览器写脚本，后端保存到 OBS 自动管理，用户无感知。这也是未来 AI 生成代码的接入点。

Phase 2：加 OBS 路径模式，给有大型项目的用户提供选择。两个 Tab 切换，互不干扰。
