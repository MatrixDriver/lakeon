设计要点

前端改动：

  • 进度卡片：追踪上传字节数计算速率，轮询后端获取处理阶段统计
  • 状态Tab：切换时带 status 参数请求后端，仅加载对应状态的文档
  • 分页：底部页码组件，每页50条，参数传给后端 API
  • 排序：点击表头切换 sort_by + sort_order 参数

后端改动：

  • GET /knowledge/documents 增加 page, page_size, status, sort_by, sort_order 参数
  • 返回增加 total_count 字段
  • 新增 GET /knowledge/documents/stats 返回各状态计数 + 处理速率
