- **SWR 组织创建**：必须在广州区域手动创建名为 `lakeon` 的组织（Namespace）。
- **SWR 登录令牌算法**：如果 Docker 标准登录失效，可使用以下算法手动生成 24 小时生效的 Token：

  - `Password`: `hmac_sha256(SecretKey, AccessKey)` 的十六进制字符串。
- **镜像版本规范**：当前全量对齐为 `0.1.0` 版本。

### 2.2 网络与安全组自动打通
- **RDS 连通性**：默认情况下，CCE 自动创建的安全组不开放 5432 端口。
- **确认有效方案**：通过调用华为云 VPC API，向安全组 `a60047cf-f803-48be-bbd0-114969044152` 自动注入了准入规则（允许 `10.11.0.0/16` 访问 TCP 5432）。
- **RDS 私网地址**：已确认稳定地址为 `10.11.0.217`。

### 2.3 确定性部署方法
- **绕过 Helm 锁**：在多次升级失败导致 Helm 状态锁定时，最有效的方案是：
  1. `helm template` 生成全量渲染后的 YAML。
  2. `kubectl apply -f` 强制提交。
- **命名空间归属修复**：若命名空间已存在导致 Helm 拒绝安装，需手动打标：
  - Label: `app.kubernetes.io/managed-by=Helm`
  - Annotations: `meta.helm.sh/release-name=lakeon`, `meta.helm.sh/release-namespace=lakeon`

## 3. 踩过的坑与失败方案 (Pitfalls)

### 3.1 镜像 Manifest 解析错误 (致命坑)
- **现象**：推送镜像到 SWR 时报 `Invalid image, fail to parse 'manifest.json'`。
- **失败原因**：现代 Docker Buildx 默认产生 **多架构 (Multi-arch) Manifest List**，而华为云 SWR 部分旧版本 Registry 节点无法解析这种格式。
- **对策**：构建时必须强制指定单平台 `--platform linux/amd64`，或者直接关闭 Buildkit 导出模式 (`DOCKER_BUILDKIT=0`)。

### 3.2 匿名 Token 401 报错
- **现象**：Pod 报 `failed to fetch anonymous token: 401`。
- **原因**：即便本地 Docker 登录成功，K8s 集群也必须拥有对应的 `swr-secret` (Type: docker-registry)。且该 Secret 中的 Token 必须通过上述 HMAC 算法生成，简单的 IAM 密码有时无效。

### 3.3 RBAC 命名空间缺失
- **现象**：部署报错 `namespaces "lakeon-compute" not found`。
- **原因**：Helm 不会自动创建 RoleBinding 交叉引用的命名空间。
- **对策**：部署前必须显式执行 `kubectl create namespace lakeon-compute`。
